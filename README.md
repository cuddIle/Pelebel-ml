# Pelebel ML — Body Pose Letter Detection

A machine learning system that recognizes Hebrew/sign-language-style letters (A, B, C, D, …) from full-body poses in real time via webcam.

It uses **MediaPipe** to extract skeletal pose landmarks, computes **joint angles** from those landmarks as features, and classifies them with a **Random Forest** model.

---

## How It Works

```
Images of poses  →  MediaPipe landmarks  →  Joint angles  →  Random Forest  →  Letter label
```

1. **Dataset** — a folder of images, one sub-folder per letter (e.g. `dataset/A/`, `dataset/B/`).
2. **Landmark extraction** — MediaPipe's pose model detects 33 body keypoints per image and saves them as JSON.
3. **Feature engineering** — 14 joint angles (elbow, shoulder, hip, knee, armpit, side-bend, leg-chain — both sides) are computed from the keypoints. Angles are rotation-invariant compared to raw coordinates.
4. **Training** — a Random Forest classifier is trained on the angle vectors and exported to both `.joblib` and `.onnx`.
5. **Inference** — live webcam feed is classified at 2 Hz; a letter is confirmed only after 5 consecutive identical predictions.

## Setup

**Requirements:** Python 3.12+, [uv](https://github.com/astral-sh/uv), [git-lfs](https://git-lfs.com)

### 1. Install git-lfs

The trained model files (`*.joblib`, `*.onnx`, `*.task`) are stored via Git LFS. Install it **before** cloning so the binaries are fetched correctly:

```bash
# macOS
brew install git-lfs

# Ubuntu / Debian
sudo apt install git-lfs

# Enable LFS for your user (one-time, machine-wide)
git lfs install
```

### 2. Clone the repo

```bash
git clone https://github.com/<owner>/Pelebel-ml.git
cd Pelebel-ml
```

If you already cloned without LFS installed, pull the binaries now:

```bash
git lfs pull
```

### 3. Install Python dependencies

```bash
uv sync
```

### 4. (Optional) Download the MediaPipe pose model

`pose_landmarker.task` is already stored in `model/` via LFS, so step 1 handles it. If for any reason it's missing:

```bash
curl -L -o model/pose_landmarker.task \
  https://storage.googleapis.com/mediapipe-models/pose_landmarker/pose_landmarker_heavy/float16/latest/pose_landmarker_heavy.task
```

### Note on the dataset

The raw image dataset (`dataset/`) and intermediate landmarks (`landmarks-dataset/`) are **not** stored in git — they're too large. Only the trained model artifacts are versioned. To re-train from scratch, you'll need to recreate the dataset locally (see "Training the Model" below).

### Working with Git LFS

After making changes to model files:

```bash
# Confirm the file is tracked by LFS (look for "filter=lfs")
git check-attr -a model/rf_classifier.joblib

# Stage, commit, and push as normal — LFS uploads happen automatically
git add model/rf_classifier.joblib
git commit -m "Update trained classifier"
git push
```

To add new file types to LFS tracking:

```bash
git lfs track "*.h5"     # example
git add .gitattributes   # commit the tracking rule
```

List everything LFS is tracking in this repo:

```bash
git lfs ls-files
```

## Training the Model

### 1. Prepare your dataset

Create one sub-folder per letter under `dataset/`, and place pose images inside:

```
dataset/
  A/  image1.jpg  image2.jpg  ...
  B/  image1.jpg  ...
```

Create the output directory for landmarks:

```bash
mkdir landmarks-dataset
```

### 2. Extract landmarks

```bash
cd src
uv run python extract_landmarks.py
```

This runs MediaPipe on every image and writes a JSON file of (x, y) coordinates to `landmarks-dataset/<letter>/`.

### 3. Train the classifier

```bash
uv run python train_model.py
```

This:
- Computes 14 joint angles per sample
- Balances classes to the same sample count
- Trains a Random Forest (1 000 trees, `min_samples_leaf=4`)
- Reports test accuracy and a per-class classification report
- Saves `model/rf_classifier.joblib`, `model/rf_classifier.onnx`, and `model/label_encoder.joblib`

### 4. Run live detection

```bash
uv run python detect_letter.py
```

Opens the default webcam. The detected letter is overlaid on the video feed along with per-letter confidence bars. Press `q` to quit.

## Why Random Forest?

The primary reason is pragmatic: **it worked well and was fast to set up**, which let us move quickly. We had a small dataset and needed something running end-to-end without a lot of iteration time. Random Forest fit that need — minimal configuration, no GPU, good accuracy out of the box.

Secondary benefits that reinforced the choice:
- Works well with small datasets and low-dimensional tabular features (14 angles)
- No GPU required — trains and infers on CPU in milliseconds
- Little tuning needed to avoid overfitting

## Future Work

Random Forest is a starting point. As the project grows the plan is to:

- **Collect more data** — more images per letter, more people, more backgrounds
- **Replace the classifier with a neural network** — once enough data is available, a small MLP or sequence model (e.g. trained on landmark sequences across frames) will handle more letters, subtle pose variations, and continuous gestures that Random Forest cannot capture well
